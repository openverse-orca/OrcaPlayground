## 🔧 手动拖入资产进行调试

手动拖动资产的通用操作方式见**项目根目录 [README - 手动拖动资产（运行前必做）](../../README.md#-手动拖动资产运行前必做)**。

为了增添多场景物理交互，请先把 G1 actor 拖到布局中，再与地形、障碍物或其他场景元素一起调整位置。

脚本启动时会扫描场景中的关节、执行器和传感器后缀，自动识别完整匹配的 G1 实例；如果匹配不完整或场景里不是目标型号，会直接报错退出。

## 运行前准备

- 场景中需要且只能有 1 台完整匹配的 G1
- 机器人实例名不需要固定，脚本会自动绑定真实名称
- `agent_name` 现在仅作兼容保留，不再决定实际绑定对象

## 运行

默认使用 **OrcaStudio 场景键盘**（无需终端焦点）：

```bash
python examples/g1/run_g1_sim.py
```

使用**运行脚本的终端**接收按键（需保持该终端窗口焦点），可加 `--keyboard console`：

```bash
python examples/g1/run_g1_sim.py --keyboard console
```

说明：
- `--keyboard orcastudio`（默认）：通过 OrcaGym gRPC 读取 OrcaStudio 场景内按键。
- `--keyboard console`：通过 `sshkeyboard` 读控制台输入；仿真仍连同一 `--orcagym-addr`（默认 `127.0.0.1:50051`）。
- 所有启动、扫描和异常信息请查看左下角**终端按钮**中的输出。

## 键盘控制

### 策略控制
| 按键 | 功能 |
|------|------|
| `1` | 启用策略控制（站立） |
| `o` | 停止策略控制 |
| `i` | 重置到初始状态 |
| `r` | 重置整段仿真 |
| `2` | 切换站立/行走模式 |

### Mimic 动作
| 按键 | 功能 |
|------|------|
| `3` | 执行/取消 Mimic 动作 |
| `4` | 切换到下一个 Mimic 动作 |
| `5` | 切换到上一个 Mimic 动作 |

### 移动控制（行走模式下）
| 按键 | 功能 |
|------|------|
| `w/s` | 前进/后退 |
| `a/d` | 左移/右移 |
| `q/e` | 左转/右转 |
| `z` | 停止移动 |

### 其他
| 按键 | 功能 |
|------|------|
| `,` / `.` | 增加/减少站立高度 |
| `6`/`7`/`8`/`9` | 粗调/细调 Kp 增益（与旧版 `4`/`5`/`6`/`7` 对应） |
| `0` | 重置 Kp 增益 |

说明：主命令使用数字键 `1`–`5`（替代原 F1–F5），便于在 `--keyboard console`（sshkeyboard）下使用；OrcaStudio 场景键盘同样按上表映射。
